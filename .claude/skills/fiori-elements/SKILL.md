# Fiori Elements Annotation Patterns

## Core Rules

1. **All annotations in `app/*/annotations.cds`** — never inline in `db/` or `srv/*.cds`
2. **`@Common.Label`** required on every element exposed to the UI
3. **`@UI.HeaderInfo`** required on every Object Page entity
4. **`@Common.ValueList`** for FK fields — no free-text input for references
5. **`@Capabilities`** to restrict operations not allowed in the UI

## File Structure

```
app/
└── orders/
    ├── annotations.cds        # All UI annotations
    ├── manifest.json          # UI5 app config (fiori:template)
    └── webapp/
        └── (generated by Fiori tools)
```

## List Report Pattern

### Minimum required annotations

```cds
using OrderService as service from '../../srv/order-service';

// Table columns
annotate service.Orders with @(
  UI.LineItem: [
    { $Type: 'UI.DataField', Value: orderNumber,  Label: 'Order #',   ![@UI.Importance]: #High },
    { $Type: 'UI.DataField', Value: customerName, Label: 'Customer',  ![@UI.Importance]: #High },
    { $Type: 'UI.DataField', Value: status,       Label: 'Status',    ![@UI.Importance]: #Medium },
    { $Type: 'UI.DataField', Value: totalAmount,  Label: 'Amount',    ![@UI.Importance]: #Low },
    { $Type: 'UI.DataField', Value: createdAt,    Label: 'Created',   ![@UI.Importance]: #Low }
  ],

  // Search filters (only indexed fields)
  UI.SelectionFields: [ status, createdAt, customer_ID ],

  // Page title
  UI.HeaderInfo: {
    TypeName: 'Order',
    TypeNamePlural: 'Orders',
    Title: { Value: orderNumber }
  }
);
```

### Status with Criticality (colored)

```cds
annotate service.Orders with @(
  UI.LineItem: [
    {
      $Type: 'UI.DataField',
      Value: status,
      Criticality: statusCriticality,      // computed element: 0=neutral, 1=error, 2=warning, 3=success
      CriticalityRepresentation: #WithIcon
    }
  ]
);
```

Add computed element in handler (`after` READ):
```typescript
srv.after('READ', 'Orders', (orders) => {
  for (const o of [orders].flat()) {
    o.statusCriticality = o.status === 'Approved' ? 3
      : o.status === 'Rejected' ? 1
      : o.status === 'PendingApproval' ? 2
      : 0;
  }
});
```

## Object Page Pattern

```cds
annotate service.Orders with @(
  UI.HeaderInfo: {
    TypeName: 'Order',
    TypeNamePlural: 'Orders',
    Title:    { $Type: 'UI.DataFieldForAnnotation', Label: 'Order #',  Value: orderNumber },
    Description: { $Type: 'UI.DataFieldForAnnotation', Label: 'Status', Value: status }
  },

  // Header facets (shown in collapsed header)
  UI.HeaderFacets: [
    {
      $Type: 'UI.ReferenceFacet',
      ID: 'HeaderGeneralInfo',
      Target: '@UI.FieldGroup#HeaderInfo',
      Label: 'General'
    }
  ],

  // Field groups (define sets of fields)
  UI.FieldGroup #HeaderInfo: {
    Data: [
      { $Type: 'UI.DataField', Value: createdAt, Label: 'Created At' },
      { $Type: 'UI.DataField', Value: createdBy, Label: 'Created By' }
    ]
  },

  // Section layout (tabs/sections on the object page)
  UI.Facets: [
    {
      $Type: 'UI.ReferenceFacet',
      ID: 'GeneralSection',
      Target: '@UI.FieldGroup#GeneralData',
      Label: 'General Data'
    },
    {
      $Type: 'UI.ReferenceFacet',
      ID: 'ItemsSection',
      Target: 'items/@UI.LineItem',
      Label: 'Order Items'
    }
  ],

  UI.FieldGroup #GeneralData: {
    Data: [
      { $Type: 'UI.DataField', Value: orderNumber, Label: 'Order Number' },
      { $Type: 'UI.DataField', Value: customer_ID, Label: 'Customer' },
      { $Type: 'UI.DataField', Value: totalAmount, Label: 'Total Amount' },
      { $Type: 'UI.DataField', Value: notes,       Label: 'Notes' }
    ]
  }
);
```

## Common Labels

```cds
annotate service.Orders with {
  orderNumber  @Common.Label: 'Order Number';
  status       @Common.Label: 'Status';
  totalAmount  @Common.Label: 'Total Amount';
  createdAt    @Common.Label: 'Created At';
  createdBy    @Common.Label: 'Created By';
  modifiedAt   @Common.Label: 'Modified At';
  customer     @Common.Label: 'Customer';
}
```

## Value Helps (FK References)

```cds
// Value help for customer field (dropdown / search help)
annotate service.Orders with {
  customer @(
    Common.ValueList: {
      CollectionPath: 'Customers',
      Parameters: [
        { $Type: 'Common.ValueListParameterOut',    LocalDataProperty: customer_ID,   ValueListProperty: 'ID'   },
        { $Type: 'Common.ValueListParameterOut',    LocalDataProperty: customerName,  ValueListProperty: 'name' },
        { $Type: 'Common.ValueListParameterDisplayOnly', ValueListProperty: 'city'   }
      ]
    },
    Common.ValueListWithFixedValues: false
  );
}
```

### Value help for status (fixed list)

```cds
annotate service.Orders with {
  status @(
    Common.ValueList: {
      CollectionPath: 'StatusValues',
      Parameters: [
        { $Type: 'Common.ValueListParameterOut',         LocalDataProperty: status,      ValueListProperty: 'code'  },
        { $Type: 'Common.ValueListParameterDisplayOnly', ValueListProperty: 'description' }
      ]
    },
    Common.ValueListWithFixedValues: true
  );
}
```

## Capabilities (Restrict Operations)

```cds
annotate service.Orders with @(
  Capabilities.InsertRestrictions: { Insertable: true },
  Capabilities.UpdateRestrictions: { Updatable: true },
  Capabilities.DeleteRestrictions: { Deletable: false }  // no delete in UI
);

annotate service.OrderItems with @(
  Capabilities.InsertRestrictions: { Insertable: true },
  Capabilities.DeleteRestrictions: { Deletable: true }
);
```

## Actions in the UI

```cds
// Action button in List Report toolbar or Object Page header
annotate service.Orders with @(
  UI.LineItem: [
    // ... columns ...
    {
      $Type: 'UI.DataFieldForAction',
      Action: 'OrderService.submitForApproval',
      Label: 'Submit for Approval',
      ![@UI.Importance]: #High
    }
  ]
);

// Action confirmation dialog
annotate service.Orders actions {
  submitForApproval @(
    Common.IsActionCritical: true,
    Core.OperationAvailable: { $edmJson: { $Path: 'in/isSubmittable' } }
  );
};
```

## Draft Handling

```cds
// In service definition
service OrderService {
  @odata.draft.enabled
  entity Orders as projection on db.Orders;
}
```

With draft enabled, Fiori Elements automatically handles the save/discard pattern. No extra annotations needed for basic draft.

## Navigation Annotations

```cds
// Semantic object navigation (cross-app)
annotate service.Orders with {
  customer @(
    Common.SemanticObject: 'Customer',
    Common.SemanticObjectAction: 'displayFactSheet'
  );
}
```

## Formatting Annotations

```cds
annotate service.Orders with {
  totalAmount @(
    Measures.ISOCurrency: currency_code,  // formats as currency
    UI.DataPoint: { Value: totalAmount, Title: 'Total Amount' }
  );

  quantity @(
    Measures.Unit: uom_code               // formats with unit of measure
  );
}
```

## Anti-Patterns

```cds
// WRONG - annotation in srv/ layer
service OrderService {
  entity Orders as projection on db.Orders {
    @Common.Label: 'Order #'   // Don't put UI annotations here
    orderNumber
  }
}

// CORRECT - annotation in app/*/annotations.cds
annotate service.Orders with {
  orderNumber @Common.Label: 'Order Number';
}


// WRONG - missing value help for FK
annotate service.Orders with {
  customer_ID @Common.Label: 'Customer';  // free text for an FK — confusing for users
}

// CORRECT - value help
annotate service.Orders with {
  customer @Common.ValueList: { CollectionPath: 'Customers', Parameters: [...] };
}
```
